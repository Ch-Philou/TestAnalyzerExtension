Rapport d'Audit de Sécurité

Cible : Extension Firefox "IA Detection: Text Analysis" (v1.0)

Date : 23 Novembre 2025

1. Analyse des Permissions (Manifest V3)

L'extension utilise le Manifest V3, qui est le standard actuel le plus sécurisé pour les extensions web.

  • host_permissions: <all_urls>
    Analyse : Cette permission est puissante car elle permet à l'extension de lire le contenu de toutes les pages visitées.
    Justification : C'est nécessaire pour le fonctionnement même de l'extension (analyse de texte sur n'importe quel site).
    Risque : Si l'extension était compromise, elle pourrait lire des données sensibles. Cependant, le code source montre qu'aucune donnée n'est exfiltrée.
  • permissions: storage
    Utilisé uniquement pour sauvegarder les préférences utilisateur (seuils, poids) localement. Aucun risque majeur.
  • permissions: scripting
    Utilisé pour injecter le script d'analyse. Standard pour ce type d'outil.

2. Confidentialité des Données (Privacy)

Excellent

L'audit du code source (content.js, background.js) révèle que :

  • Aucune requête réseau externe : L'extension ne communique avec aucun serveur (pas de Google Analytics, pas de télémétrie, pas d'API tierce).
  • Traitement Local : Toute l'analyse de texte (calcul d'entropie, détection de connecteurs) se fait à 100% dans le navigateur de l'utilisateur.
  • Pas de stockage de données sensibles : Le texte analysé n'est jamais stocké, il est traité à la volée et oublié dès la fermeture de la page ou de la modale.

3. Sécurité du Code (XSS & Injection)

Injection de Contenu (innerHTML)

Le fichier content.js utilise innerHTML pour construire l'interface de la modale.

  • Vecteur potentiel : Si une variable injectée contenait du code malveillant (ex: <script>...), cela pourrait exécuter du code (XSS).
  • Audit des variables :
    • t.title, t.labels : Proviennent de l'objet I18N (codé en dur dans le fichier). Sûr.
    • scores : Ce sont des nombres calculés mathématiquement. Sûr.
    • langName : Provient de la liste LANG_NAMES (codée en dur). Sûr.
  • Conclusion : Bien que l'utilisation de innerHTML soit généralement déconseillée, dans ce contexte précis avec des données contrôlées, le risque est nul.

Communication (Message Passing)

Les messages entre le popup, le background et le content script (show_results, recalculate) sont simples et ne transportent pas de charge utile complexe (payload) pouvant être exploitée.

4. Recommandations & Corrections

L'état actuel est sécurisé pour un usage en production.

Améliorations futures (Best Practices) :

  1. Sanitization : Si vous prévoyez d'afficher des données provenant de la page (ex: "Extrait du texte analysé : ..."), il faudra impérativement "échapper" (sanitize) ce texte avant de l'insérer via innerHTML pour éviter qu'un site malveillant ne piège l'extension.
  2. CSP (Content Security Policy) : Le Manifest V3 impose déjà une CSP stricte qui empêche l'exécution de code distant, ce qui est une excellente protection par défaut.

Conclusion

L'extension IA Detection respecte les standards de sécurité et de confidentialité.

Elle est "Privacy-First" (tout est local) et ne présente pas de vulnérabilité critique exploitable dans sa version actuelle.
